### Corrections Critiques

#### 1. exposeStores.js - Singleton mal utilisé
**Problème** : Tentative d'instanciation avec `new ProjectSyncAdapter()` alors que le module exporte déjà une instance singleton.

**Solution** :
```javascript
// ❌ Avant
const projectSyncAdapter = new ProjectSyncAdapter();

// ✅ Après
// ProjectSyncAdapter est déjà une instance singleton exportée
window.__SYNC_TOOLS__ = {
  cleanupOrphanedProjects: () => ProjectSyncAdapter.cleanupOrphanedProjects(),
  collectAllStoreData: () => ProjectSyncAdapter.collectAllStoreData()
};
```

**Fichier** : `src/utils/exposeStores.js:83`

#### 2. ScriptoriumRoom.jsx - Hook useTheme manquant
**Problème** : Référence à `theme.colors.accents.warm` sans import ni appel du hook.

**Solution** :
```javascript
// Ajout import
import { useTheme } from "styled-components";

// Appel du hook
const ScriptoriumRoom = () => {
  const theme = useTheme();
  // ...
```

**Fichier** : `src/components/rooms/Scriptorium/ScriptoriumRoom.jsx:42`

### Audit ESLint Complet

#### Configuration ESLint améliorée
Création d'une configuration séparée pour scripts Node.js vs code React/Browser :

**Fichier** : `eslint.config.js`
```javascript
export default defineConfig([
  globalIgnores(['dist']),
  // Configuration pour les scripts Node.js
  {
    files: ['scripts/**/*.js'],
    languageOptions: {
      globals: { ...globals.node }
    }
  },
  // Configuration pour les fichiers React/Browser
  {
    files: ['src/**/*.{js,jsx}'],
    extends: [
      js.configs.recommended,
      reactHooks.configs['recommended-latest'],
      reactRefresh.configs.vite,
    ],
    languageOptions: {
      globals: globals.browser
    }
  }
])
```
